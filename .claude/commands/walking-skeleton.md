# Walking Skeleton

## Problem

As an AI assistant, you might focus on perfecting individual components before connecting them together, which can lead to components that don't integrate well or meet actual requirements.

## Example Scenarios

- You help develop microservices without suggesting cross-service integration testing
- You create UI components without connecting them to backend APIs
- You implement data processing functions without showing the full data flow
- You develop features in isolation without testing them in the full application context
- You focus on optimizing individual components before proving the overall architecture works

## Recommended Actions

1. **Start with end-to-end**: Suggest implementing a minimal but complete version of the system that includes all necessary components connected together.

2. **Accept initial limitations**: Propose a first implementation that may be basic or imperfect as long as it demonstrates the full workflow.

3. **Emphasize real integration**: Recommend ensuring all components can communicate and work together before focusing on perfecting any single piece.

4. **Suggest iteration based on usage**: Once the end-to-end system works, advise using insights gained from actual usage to prioritize improvements.

5. **Offer rapid prototyping**: Use your capabilities to quickly build the initial skeleton implementation across all components.

## Anti-patterns to Avoid

- Perfecting individual components in isolation
- Not suggesting integration testing until late in development
- Creating features without explaining how they integrate with the whole system
- Adding unnecessary complexity before establishing basic functionality
- Focusing on optimization before establishing the correct architecture

## Remember

When helping users build systems, prioritize getting a functioning end-to-end skeleton working quickly over perfecting individual components. A rudimentary but complete system provides much more valuable feedback than perfectly polished but disconnected components.
